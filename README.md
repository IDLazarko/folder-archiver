Скрипт для автоматической архивации папок второго уровня вложенности. Находит все папки, находящиеся на втором уровне иерархии, архивирует их в ZIP и перемещает в корневую папку.

## ✨ Возможности

- 📂 **Автоматический поиск** папок второго уровня вложенности
- 🗜️ **Сжатие в ZIP** с оптимизацией размера
- 📊 **Прогресс-бары** для отслеживания процесса
- 📈 **Статистика сжатия** с отображением экономии места
- 🧹 **Очистка структуры** - удаление пустых родительских папок
- 🔍 **Предварительный просмотр** перед началом операции
- 🛡️ **Обработка ошибок** с сохранением данных
- 💾 **Читаемые размеры** файлов (B, KB, MB, GB, TB)

## 📦 Требования

- Python 3.6 или выше
- Установленные библиотеки:
  - `tqdm` - для прогресс-баров
  - `humanize` - для читаемого форматирования размеров

## 🚀 Использование

### Базовое использование

```bash
# Архивировать папки в текущей директории
python archive_folders_v2.py

# Архивировать папки в указанной директории
python archive_folders_v2.py /путь/к/папке
```

### Расширенное использование

```bash
# Не удалять оригинальные папки после архивации
python archive_folders_v2.py --keep-original

# Архивировать также пустые папки
python archive_folders_v2.py --include-empty

# Перезаписывать существующие архивы
python archive_folders_v2.py --overwrite

# Пропустить предварительный просмотр
python archive_folders_v2.py --no-preview

# Минимизировать вывод (только прогресс-бары)
python archive_folders_v2.py --quiet

# Комбинация параметров
python archive_folders_v2.py /путь/к/папке --keep-original --include-empty
```

## 🎛️ Параметры командной строки

| Параметр | Описание | Значение по умолчанию |
|----------|----------|----------------------|
| `path` | Путь к корневой папке | Текущая папка (.) |
| `--keep-original` | Не удалять оригинальные папки | False (удалять) |
| `--include-empty` | Архивировать пустые папки | False (пропускать) |
| `--overwrite` | Перезаписывать существующие архивы | False (пропускать) |
| `--no-preview` | Пропустить предварительный просмотр | False (показывать) |
| `--quiet` | Минимизировать вывод | False (подробный) |
| `--help` | Показать справку | - |

## 📁 Структура архивации

### Именование архивов

Архивы именуются по формату: `{родительская_папка}_{имя_папки}.zip`

Примеры:
- `Projects_Python.zip`
- `Photos_Vacation2023.zip`
- `Documents_Reports.zip`

### Содержимое архивов

Внутри архива сохраняется структура оригинальной папки:

```
Projects_Python.zip
├── script1.py
├── script2.py
└── README.md

Photos_Vacation2023.zip
├── photo1.jpg
├── photo2.jpg
└── photo3.jpg
```

## 📊 Логирование и отчеты

Скрипт предоставляет подробную информацию на каждом этапе:

1. **Предварительный просмотр** - список всех найденных папок
2. **Прогресс архивации** - текущее состояние с процентами
3. **Статистика сжатия** - размер до/после, процент сжатия
4. **Итоговый отчет** - общая статистика по операции
5. **Список архивов** - созданные файлы с размерами

## 🛡️ Обработка ошибок

Скрипт безопасно обрабатывает различные ошибки:

- **Отсутствие прав доступа** - пропускает папку и продолжает работу
- **Поврежденные файлы** - удаляет неполный архив
- **Недостаточно места** - выводит предупреждение
- **Конфликты имен** - проверяет существующие архивы

## ❓ Часто задаваемые вопросы

### Q: Что произойдет, если архив с таким именем уже существует?
A: Скрипт пропустит эту папку и продолжит работу с остальными (если не указан флаг `--overwrite`).

### Q: Можно ли восстановить оригинальную структуру?
A: Да, распаковав архивы. Имена архивов содержат информацию о родительских папках.

### Q: Что будет с пустыми родительскими папками?
A: После архивации всех дочерних папок пустые родительские папки автоматически удаляются.

### Q: Как отменить операцию?
A: На этапе предварительного просмотра можно ответить "n" для отмены.

### Q: Поддерживаются ли другие форматы архивов?
A: В текущей версии только ZIP. Другие форматы могут быть добавлены позже.
